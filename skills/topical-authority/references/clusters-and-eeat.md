# Content clusters & E-E-A-T — the framework

## Content clusters (topical depth)

Structure: one **pillar** + many **supporting** pages, interlinked.

```
            ┌─────────────────────────┐
            │  PILLAR: <core topic>   │  ← definitive overview, links out & in
            │  e.g. "Stablecoin       │
            │  payments: the guide"   │
            └───────────┬─────────────┘
        ┌───────────────┼───────────────┐
   [What is USDC]   [USDC vs USDT]   [How to accept
    (glossary)      (comparison)      stablecoins] (how-to)
        └──────── all link to pillar + each other ────────┘
```

- **Pillar**: broad, authoritative, covers the whole topic; the page you most
  want cited for the head term.
- **Supporting pages**: one per sub-question / long-tail (from your
  `prompt-coverage` map). Each answer-first; each links up to the pillar and
  across to siblings.
- The signal to engines: "this site covers <topic> comprehensively" → authority.

Build one cluster per core topic you must own. Depth beats scattered one-offs.

## E-E-A-T signals (demonstrated trust)

Engines weigh **Experience, Expertise, Authoritativeness, Trust**. Make each real
and visible:

- **Authors**: bylines on substantive content; **bio pages** with real
  credentials; `author` / `Person` schema linked to the article.
- **Experience**: first-hand proof — "we tested", original screenshots, real
  data, case studies. Generic rehash reads as low-experience.
- **Expertise**: accurate, current, technically correct; cite **primary sources**.
- **Authoritativeness**: a consistent entity, third-party mentions
  (`earn-mentions`), Wikidata, recognized authors.
- **Trust**: clear **About**, **Contact**, policies; `Organization` schema; HTTPS;
  no deceptive patterns; correct, honest claims.

## Checklist

- [ ] Core topic(s) chosen; a pillar page exists for each.
- [ ] Supporting pages cover the sub-questions; all interlinked to the pillar.
- [ ] Substantive pages have real author bylines + bio pages + schema.
- [ ] First-hand experience signals (data/tests/cases), not generic content.
- [ ] Claims sourced to primaries; facts current.
- [ ] About / Contact / policy / Organization schema present and consistent.
- [ ] Entity consistent across the web (Wikidata, profiles, `sameAs`).

## The line

Authority is **earned**: real depth, real authors, real experience. Fabricated
credentials, fake "expert reviewed" badges, or invented case studies are
detectable, against guidelines, and corrode the exact trust you're building.
